# 实现说明

## 当前版本范围

本次实现已从纯样例数据版本进化为“公开真实数据优先，失败时自动回退样例数据”的可研究版本，目标是打通“真实数据 → 策略研究 → 当日指示 → API → Web 看板”的完整链路。

## 已实现内容

### 后端 API
- `GET /api/health`：健康检查
- `GET /api/dashboard`：返回完整看板数据
- `GET /api/dashboard/live`：返回基于公开真实数据和策略研究的完整看板数据
- `GET /api/fund-scores`：返回基金评分结果
- `GET /api/nav-estimates`：返回基金净值变化估算结果
- `GET /api/portfolio-advice`：返回组合建议
- `GET /api/backtest-summary`：返回回测摘要
- `GET /api/stocks`：返回样例股票数据

### 算法能力
- 基于净值序列的动量、回撤、波动率评分
- 基于基金历史净值的参数搜索与最优策略研究
- 基于滚动窗口的 walk-forward 策略验证
- 基于持仓映射、股票实时涨跌与 ETF 基准的净值变化估算
- 基于回测摘要的分数凯利仓位控制
- 单基金仓位上限控制
- 当日研究指示输出

### 持久化与缓存
- 实时看板快照已支持本地文件缓存
- 缓存目录：`apps/api/data`
- 默认缓存最近一次实时研究结果，减少重复抓数和重复回测

### 真实数据接入
- 东方财富股票历史日线接口
- 东方财富 ETF 历史日线接口
- 东方财富股票实时行情接口
- 天天基金历史净值脚本接口
- 天天基金基金估值接口

### 前端页面
- 总览头图
- 评分表格
- 策略研究曲线
- 组合建议饼图
- 净值估算柱状图
- 净值历史曲线
- 当日研究指示表格
- 风险提示与评分依据

## 当前限制
- 公开接口不是官方 SLA 数据服务，存在字段变化或限流风险
- 基金持仓结构无法做到真实实时，只能基于披露持仓与估值脚本代理
- 回测仍是简化版研究引擎，尚未纳入全部申赎细则
- 当前仅做文件缓存，尚未接入正式数据库
- 未实现用户登录、参数持久化、异步任务系统

## 开发运行说明

### 后端开发
- 命令：`npm run dev:api`
- 说明：后端使用 `tsx watch src/server.ts` 运行，控制台打印 `fund-quant-api listening on http://localhost:3000` 即表示启动成功。

### 前端开发
- 命令：`npm run dev:web`
- 说明：前端使用 Vite 开发服务器。

### 关于终端中显示 Exit Code 1
- 当前上下文中，`dev:api` 启动日志已显示服务成功监听端口。
- 若编辑器终端显示 `Exit Code: 1`，更可能是终端会话切换、中断或开发服务被外部结束，而不是编译失败。
- 代码层面的测试和构建已通过，可优先以实际启动日志和页面访问结果为准。
