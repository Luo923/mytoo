# 基金量化研究与策略辅助平台 Spec

> 本文件为实现阶段引用的规格说明，详细内容以后续迭代更新为准。

## 当前实现目标
- 从零搭建可运行的全栈研究平台
- 接入公开真实市场数据，失败时自动回退样例数据
- 提供基金评分、净值估算、策略研究、组合建议与当日研究指示 API
- 提供 Web 看板展示评分、策略曲线、组合建议与当日信号
- 使用本地缓存降低重复抓数与重复研究成本

## 一期实现边界
- A 股股票、ETF、公募场外基金
- 东方财富/天天基金公开数据接入
- 基金历史净值研究与净值变化估算
- 基金评分、基础回测、walk-forward 滚动验证
- 分数凯利资金管理与目标仓位建议
- 当日研究指示输出
- 本地文件缓存
- 不接入真实交易、不自动下单、不承诺收益

## 当前系统形态

### 后端
- 技术栈：`Node.js + TypeScript + Express`
- 目录：`apps/api`
- 作用：抓取公开数据、执行量化研究、输出 API

### 前端
- 技术栈：`React + Vite + TypeScript + Recharts`
- 目录：`apps/web`
- 作用：展示研究看板、策略曲线、仓位建议与当日研究指示

### 缓存
- 目录：`apps/api/data`
- 作用：缓存实时研究快照，避免频繁请求公开接口

## 已实现接口

- `GET /api/health`
- `GET /api/dashboard`
- `GET /api/dashboard/live`
- `GET /api/fund-scores`
- `GET /api/nav-estimates`
- `GET /api/portfolio-advice`
- `GET /api/backtest-summary`
- `GET /api/stocks`

## 当前数据来源

### 公开真实数据
- 股票历史/实时：东方财富公开接口
- ETF 历史：东方财富公开接口
- 基金历史净值与扩展信息：天天基金 `pingzhongdata`
- 基金当日估值：天天基金 `fundgz`

### 回退数据
- 若公开接口抓取失败，系统自动使用内置样例数据保证流程可运行

## 当前算法能力

### 基金评分
- 动量
- 回撤控制
- 波动率控制
- 基准强度
- 持仓共振

### 基金净值估算
- 持仓股票代理涨跌
- ETF/基准走势修正
- 天天基金估值融合

### 策略研究
- 参数搜索
- 动量筛选
- TopN 组合选择
- walk-forward 滚动验证

### 资金管理
- 分数凯利
- 单标的仓位上限
- 总仓位约束

### 当日输出
- 优先关注
- 持有观察
- 逢高减仓观察
- 回避观察

## 当前限制

- 场外基金盘中净值不是官方实时净值，只能做研究估算
- 基金持仓存在披露滞后，无法做到真实实时持仓
- 当前只做文件缓存，尚未接入正式数据库
- 回测尚未纳入完整申赎规则、费率、确认时点和暂停申赎约束
- 当前未实现登录、权限、参数持久化、异步任务调度

## 下一阶段目标

1. 引入 SQLite 或正式数据库
2. 归档历史研究结果与每日信号
3. 增强基金估值误差校准模型
4. 增强场外基金回测执行规则
5. 增加回测配置页与基金详情页
